# Unitテストから学ぶ極意
基本は「テスト対象クラス1つにつき、テストクラス1つ」
* どのクラスのテストか一目で分かる
* 修正時に迷わない
* 責任範囲が明確

「1クラス1メソッド」にする必要はないが、  
責務が1つならOK。責務が増えたら分ける
